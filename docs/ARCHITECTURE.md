## 🔐 安全考虑

1. **API Token 管理（.env 文件）**
   - Token 存储在后端 `.env` 文件中
   - `.env` 已加入 `.gitignore`，不会上传 GitHub
   - 绝不将 Token 写入前端代码、日志或前端存储

2. **文件上传安全**
   - 限制文件大小（建议 50MB）
   - 仅允许 .xlsx, .xls, .csv 格式
   - 上传文件按会话隔离存储

3. **会话安全**
   - 使用 UUID 作为会话标识
   - 会话数据内存存储，重启后清空
   - 每个会话独立管理文件和数据

4. **代码执行安全**
   - PandasAI 生成的代码在沙箱环境执行
   - 设置代码执行超时（建议 30 秒）
   - 限制可导入的模块

## ⚙️ 环境变量配置

后端使用 `.env` 文件配置 LLM（示例）：

```bash
# backend/.env
DASHSCOPE_API_KEY=sk-xxxxxxxxxxxxxxxx
DASHSCOPE_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1
DASHSCOPE_MODEL=deepseek-r1
```

项目包含 `.env.example` 模板（不含敏感信息，可提交到 GitHub）。

## 🌐 部署架构

### 开发环境
```
localhost:8080 (前端) ──► localhost:8000 (后端)
```

### 生产环境建议
```
                    ┌─────────────────┐
                    │   Nginx/CDN     │
                    │   (静态资源)    │
                    └────────┬────────┘
                             │
              ┌──────────────┴──────────────┐
              │                              │
     ┌─────────▼─────────┐        ┌─────────▼─────────┐
     │   前端静态资源     │        │   FastAPI 后端    │
     │   (index.html)     │        │   (uvicorn)       │
     └────────────────────┘        └─────────┬─────────┘
                                             │
                                    ┌────────▼────────┐
                                    │   PandasAI      │
                                    │   + DeepSeek    │
                                    └─────────────────┘
```

### Docker 部署（可选）
```dockerfile
# 后端 Dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY backend/ ./backend/
COPY pyproject.toml .
RUN pip install -e .
EXPOSE 8000
CMD ["uvicorn", "backend.main:app", "--host", "0.0.0.0", "--port", "8000"]
```
